Under the Hood

The technical detail

For those who want to see the actual implementation.

Inventory_from_SharePoint ‚Äî Power Query M

The inventory query connects to SharePoint, finds the most recently modified .xlsm file in the configured folder, loads it, and processes it ‚Äî all driven by settings cells so no hardcoding is needed.

AllFiles = SharePoint.Files(SiteURL, [ApiVersion=15]), Filtered = Table.SelectRows(AllFiles, each Text.EndsWith([Name], ".xlsm")), Sorted = Table.Sort(Filtered, {{"Date modified", Order.Descending}}), Latest = Sorted{0}[Content], ExcelFile = Excel.Workbook(Latest, null, true),

After loading, on-hand quantities are summed across all warehouse locations per item using Table.Group ‚Äî so if an item appears in multiple subinventories, the total is what flows through to the dashboard.

Grouped = Table.Group(Typed, {"Item Number"}, {{"On-Hand Qty", each List.Sum([#"On-Hand Qty"]), type number}, ...})
Excel Formula Logic ‚Äî Output & Summary Tabs

The rolling balance formula subtracts each day's total consumption (across all 4 products) from the previous day's closing balance. Build quantities are locked to their schedule rows using mixed references.

=G10-(B10*H$4)-(C10*H$5)-(D10*H$6)-(E10*H$7)

On-hand quantity is live-pulled from the Inventory Data tab via VLOOKUP, replacing the previous hardcoded values entirely.

=IFERROR(VLOOKUP(A10,'Inventory Data'!$A:$C,3,0),0)

Days of Supply uses average daily consumption across the 14-day horizon. The BOM quantities are pulled from Work Def using SUMPRODUCT to match item + product ‚Äî no fragile cell references.

=IFERROR( IF(D4="‚àû","No stockout", Output!$I$2 + D4), "No stockout")
VBA ‚Äî RefreshAllCTSData Macro

The macro iterates through ThisWorkbook.Connections and calls .Refresh on each Power Query connection by name. It updates a live timestamp on the Dashboard and shows a confirmation dialog on completion.

Sub RefreshAllCTSData() Call RefreshConnection("BOM_from_SharePoint") Call RefreshConnection("Inventory_from_SharePoint") ws.Range("A23").Value = "Last refreshed: " _ & Format(Now(), "DD-MMM-YYYY HH:MM") MsgBox "Data refreshed successfully!" End Sub

Error handling via On Error Resume Next ensures the macro degrades gracefully if a connection fails (e.g. no network), rather than crashing mid-refresh.

Data Flow ‚Äî Sheet Architecture

The workbook has a deliberate single-direction data flow. Raw sources feed into intermediate tabs, which feed the calculation engine, which feeds the outputs. No circular dependencies, no hidden cross-references.

The Settings tab acts as a configuration layer ‚Äî all SharePoint URLs, folder paths, and column name mappings live there as named ranges, referenced by Power Query M scripts. Changing a setting cell propagates through the entire system on next refresh.
